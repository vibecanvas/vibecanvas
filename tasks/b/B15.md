# B15 - canvas query subtree-not-found error prints canvas name as `undefined`

## Plan
- Reproduce the missing-subtree error after deleting a real group from the dev canvas.
- Trace how the subtree validation path formats its error message.
- Ensure missing-group query errors include the actual selected canvas name/id consistently.

## TODOS
- [x] Reproduce the bad error message with `query --subtree <deleted-group-id>` against a real canvas.
- [x] Find where `CANVAS_QUERY_SUBTREE_NOT_FOUND` is built and why it interpolates `undefined` for the canvas name.
- [x] Add coverage so missing subtree/group errors include the real canvas name consistently.

## NOTES
- Category: bug.
- Status: closed.
- Found during real dev experimentation after `canvas delete` successfully removed a group and its descendants.
- Root cause: subtree/group validation formatted the error message from `doc.name`, but some persisted docs can be missing that field even when the selected canvas row has the correct name.
- Fix: `packages/canvas-cmds/src/scene-query-shared.ts` now falls back to the selected canvas row name when `doc.name` is missing/blank, and `packages/canvas-cmds/src/cmd.query.ts` passes the selected row name into validation.
- Added regression coverage in `packages/canvas-cmds/tests/cmd.query.test.ts`.
