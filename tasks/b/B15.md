## NOTES
- Category: bug.
- Found during real dev experimentation after `canvas delete` successfully removed a group and its descendants.
- Repro sequence:
  1. delete group `87bccbb2-30d8-4dec-8f04-195a857309d0` from canvas `hello`
  2. query that deleted group as subtree root
- Actual error:
  - `Subtree root '87bccbb2-30d8-4dec-8f04-195a857309d0' was not found in canvas 'undefined'.`
- Expected error should mention the selected real canvas name, here `hello`.
- The command still uses the correct canvas id in the payload, so the bug appears to be only in message formatting/context propagation.
