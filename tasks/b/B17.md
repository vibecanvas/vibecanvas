## NOTES
Category: bug.

### Repro
```bash
bun run apps/cli/src/main.ts canvas add --schema rect
```

### Actual
```text
Add requires exactly one element source: --element, --elements-file, --elements-stdin, or shorthand flags.
Hint: Pass exactly one element source: --element, --elements-file, --elements-stdin, or shorthand flags.
Next: Try: vibecanvas add --canvas <canvas-id> --rect 10,20,120,80 --json
```

Exit code:
- `1`

### Expected
`--schema rect` should print the rect schema block and exit successfully.
It should not require:
- `--canvas`
- `--canvas-name`
- `--element`
- `--elements-file`
- `--elements-stdin`
- shorthand element flags

### Why this is bad
`--schema` reads like an info mode, not a mutation mode.
Human and agent users expect schema/help output to bypass normal add validation.
Current behavior makes discovery worse right when user asks for guidance.
