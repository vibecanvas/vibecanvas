- Confirm whether the command exits through the shared JSON error contract or crashes through Bun's fatal error path.
- Trace where document-open failures bypass structured CLI error formatting.

## NOTES
- Category: bug.
- Status: open.
- Found during the divergent real-dev canvas CLI experiment with token `aad66061`.
- `canvas list --json` succeeded, but every attempted `canvas query --json` on listed canvases failed before producing JSON.
- Expected behavior: when `--json` is passed, query failures should emit a structured JSON payload such as `{ ok: false, command, code, message, ... }`.
- Actual behavior: Bun prints a fatal stack trace from ORPC client decoding and exits with code 1.
- This is a warning/output-contract bug on top of the underlying document-unavailable issue because machine consumers cannot safely parse the result.
