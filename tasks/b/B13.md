## NOTES
- Category: bug.
- Reproduced from the real dev environment with `bun run apps/cli/src/main.ts ...`, not the isolated test harness.
- `canvas list --json` returned 5 canvases from `local-volume/data/vibecanvas.sqlite`.
- `canvas query --canvas-name hello --json` works.
- `canvas query --canvas-name ok --json` fails with `CANVAS_QUERY_FAILED` / `Document automerge:... is unavailable`.
- The same unreadable-doc failure also reproduces for `power`, `test ground`, and `recorder`.
- `canvas patch` works on `hello`, but at least twice returned `CANVAS_PATCH_FAILED` with `Timed out waiting for persisted canvas doc ...` even though an immediate follow-up `query` showed the mutation had actually landed.
- One real repro of the false-negative patch path was adding a new style key such as `strokeColor` to an existing rect.
- Likely investigation areas:
  - mismatch between canvas SQL rows and where the corresponding Automerge docs are actually stored/read from;
  - missing live fallback / repo resolution for some real docs;
  - persisted-doc verification using overly strict object equality / key-order-sensitive comparison after patch writes.

### LOGS
- Observed real-db command behavior with:
  - `bun run apps/cli/src/main.ts canvas list --json`
  - `bun run apps/cli/src/main.ts canvas query --canvas-name hello --json`
  - `bun run apps/cli/src/main.ts canvas query --canvas-name ok --json`
  - `bun run apps/cli/src/main.ts canvas patch --canvas-name hello --id cf36ae0d-2f96-489f-aaaa-f3549c5ef366 --patch '{"element":{"style":{"strokeColor":"#f59e0b"}}}' --json`
- Confirmed the false-negative patch case by running an immediate follow-up `query --id ... --json`, which showed the supposedly failed patch had persisted.
