## Plan
- Reproduce the drag latch around iframe interaction inside the canvas.
- Identify whether pointerdown is reaching the canvas/widget drag layer before focus transfers into the iframe.
- Fix the stuck drag state so pointer release always clears without requiring a canvas reload.

## NOTES
- Category: bug.
- User report: when clicking inside an iframe embedded in the canvas, pointer down appears to initiate a drag state. Moving the pointer then moves the whole iframe/browser widget, and release does not clear the drag. The only recovery is reloading the entire canvas.
- Reproduction is currently unclear / unstable.
- Likely area: `packages/canvas/src/plugins/IframeBrowserWidget.plugin.tsx` plus shared hosted-widget drag/pointer coordination.
- Important symptom: pointer capture or drag state cleanup appears to get lost once interaction crosses into the iframe content.
