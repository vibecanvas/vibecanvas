# A21 - canvas CLI: add `--dry-run` for add/patch/move/group/ungroup/delete

## Plan
Add one shared preview mode for canvas mutation commands. Dry run must validate inputs, resolve selectors, compute target changes, and print the same shape of result metadata without mutating Automerge/doc/db/files.

## TODOS
- [ ] Define exact `--dry-run` contract for `add`, `patch`, `move`, `group`, `ungroup`, `delete`
- [ ] Keep text and `--json` output explicit that no mutation happened
- [ ] Reuse command execution path as far down as possible before side effects
- [ ] Include matched ids and predicted changed ids in dry-run output
- [ ] For `delete`, report doc/effects impact separately
- [ ] Add tests for each supported subcommand in success and failure modes
- [ ] Document examples in command help

## NOTES
- Scope is canvas CLI mutation commands only: `add`, `patch`, `move`, `group`, `ungroup`, `delete`
- Dry run should still fail on invalid flags, ambiguous canvas selection, missing ids, invalid patch payloads, and other normal validation errors
- Dry run must not write Automerge state, DB rows, files, images, or side-effect transports
- Prefer one shared flag name and one shared result field like `dryRun: true`
- Good default: output as if command succeeded, but with `changedCount`/`addedCount`/`deleted*` predicted only
- Watch for commands with mixed doc and external effects, especially `delete`

### LOGS
- Added task from user request.
- Implemented `--dry-run` plumbing for canvas CLI mutation commands: add/patch/move/group/ungroup/delete.
- Dry-run now validates normally, returns `dryRun: true`, and skips Automerge/doc mutation + flush.
- `add --dry-run` now returns placeholder ids as `PLACEHOLDER-NO` instead of real generated ids.
- Added CLI tests covering dry-run success paths for all supported mutation subcommands.
