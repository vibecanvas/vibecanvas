# A8 - canvas CLI: `inspect` command

## Plan
- Add exact-id inspection for one element or group inside one selected canvas.
- Keep the command readonly and precise.
- Reuse the shared output modes `summary`, `focused`, and `full`.

## NOTES
- Category: addition.
- Scope decision: final naming is `inspect`, not `show`.
- Dependency: build after A5 and A6.
- Actual CLI e2e coverage lives in `apps/server/tests/cli/`, which is where the A6 harness landed.
- `--canvas-name` resolves via unique case-insensitive substring matching; 0 matches and many matches both fail instead of guessing.
- `summary` returns compact metadata, `focused` adds target-specific details, and `full` returns the raw persisted record inside the stable inspect envelope.
- Group summary output now also includes `directChildElementIds` and `directChildGroupIds`, so the default text/JSON inspect result is still compact but no longer hides immediate group membership.
- `--json` success payload shape is `{ ok, command, mode, selector, canvas, target }`; JSON failures stay on stderr with stable `code` values.
- The inspect command suppresses migration/automerge startup logs while loading offline state so stdout stays contract-safe for both text and JSON modes.
- `vibecanvas canvas inspect --help` now serves a command-specific help screen with selector rules, shared `--db` fallback behavior, output modes, and examples.
