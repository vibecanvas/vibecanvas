# A31 - theme: remove Tailwind from frontend and stop scanning canvas sources
[Overview](../BASED.md)

Finish Tailwind removal in frontend after canvas package ships its own CSS.

## Context
Frontend still imports Tailwind in `apps/frontend/src/index.css`, uses Tailwind utility classes throughout app UI, and scans `packages/canvas/src` via `@source`. Once canvas package is decoupled, frontend can move to plain CSS and shared theme vars.

Relevant files:
- `apps/frontend/src/index.css`
- `apps/frontend/vite.config.ts`
- `apps/frontend/package.json`
- `apps/frontend/src/App.tsx`
- `apps/frontend/src/components/`
- `apps/frontend/src/feature/sidebar/components/`

## Plan
### 1. Remove canvas source scanning from frontend
Delete Tailwind `@source` dependency on canvas package source.

### 2. Migrate frontend UI to plain CSS
Replace utility classes with semantic frontend classes and CSS rules.

### 3. Remove Tailwind build/runtime deps
Delete Tailwind Vite plugin and packages after UI no longer depends on them.
