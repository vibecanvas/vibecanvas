# A9 - canvas CLI: `query` command

## Plan
- Add structured scene queries without any natural-language parsing.
- Support explicit selectors so agents can ask for only the scene slice they need.
- Keep the command readonly and machine-friendly.

## TODOS

- [x] Implement `vibecanvas canvas query` with structured flags plus either `--where` or `--query`, rejecting ambiguous combinations.
- [x] Return stable match metadata and scene payloads in `--json`.
- [x] Keep the command strictly readonly with no natural-language mode.

## NOTES
- Category: addition.
- Scope decision: `query` stays structured and readonly in v1.
- Dependency: build after A5 and A6.
- Actual CLI e2e coverage lives in `apps/server/tests/cli/`, which is where the A6 harness landed.
- `query` now owns the readonly scene lookup surface, including the exact-id case that previously had a dedicated `inspect` command.
- Selector input styles are mutually exclusive: use structured flags, `--where`, or `--query`, but not more than one style at once.
- `--where` is a URL-query-string style selector (`type=rect&type=text&subtree=group-root`), while `--query` accepts structured JSON with the same filter fields.
- `query` also supports exact persisted style matching via repeated `--style key=value` flags, `style.<key>=...` entries inside `--where`, and a `style` object inside `--query`.
- `--group` matches direct children only; `--subtree` includes the root group and all nested descendants.
- Bounds filters use computed persisted geometry, and group bounds are derived from descendant elements; empty groups do not match bounds selectors.
- `--json` success payload shape is `{ ok, command, mode, selector, canvas, count, matches[] }`, with each match split into stable `metadata` and output-mode-specific `payload` fields; query payloads now always include `data` and `style` subfields (`null` for groups).
- Default summary text output now also prints `data=...` and `style=...` for every match so non-JSON query output stays machine-friendly.
- Query now also supports `--omitdata` and `--omitstyle` so callers can drop those subfields from both text and JSON output when they do not need them.
- Top-level CLI help now explicitly teaches subcommand-level `--help` discovery and documents that canvas subcommands can expose help through unprefixed aliases like `vibecanvas query --help`.
- `vibecanvas canvas query --help` and the top-level `vibecanvas canvas --help` output now document query selector syntax and the readonly/no-natural-language constraint.
