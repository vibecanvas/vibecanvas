# A20 - canvas CLI: agent-friendly help, discovery, and forgiving errors

## Plan
Current canvas CLI is usable for humans with source access, but rough for AI agents probing it blind. Research from `llm/reports/canvas-cli-dev-report-2026-04-07.md` found three bad patterns:
- error messages can be technically true but not discovery-friendly (`Unknown patch branch 'locked'`)
- online/no-`--db` path can briefly disagree with offline `--db` path after recent mutations

Goal:
- make the CLI forgiving for common agent mistakes
- help agents discover the surface without reading source
- keep help incremental so one command does not dump huge context

Scope:
- help and error-contract pass over the canvas CLI first
- improve discovery, suggestions, examples, and next-step hints
- do not expand into natural-language parsing

## NOTES
- Main evidence so far lives in `llm/reports/canvas-cli-dev-report-2026-04-07.md`.
- Concrete issues already seen:
  - patch help does not explain the envelope shape, so a natural payload like `{"locked":true}` fails with low-discovery output.
  - no-`--db` query can briefly show stale structure after offline `--db` mutations.
- Good CLI for agents should prefer:
  - exact error code
  - short explanation
  - one likely fix
  - one minimal example
- Keep help small. Do not dump every arg on every failure.
- Good first slice: fix help routing + patch hints + unknown-command suggestions before bigger forgiveness rules.
