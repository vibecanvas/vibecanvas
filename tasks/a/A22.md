# A22 - canvas CLI: allow JSON array payloads for multi-element add and multi-target patch

## Plan
Keep JSON as the authoring format. Improve batch authoring by letting CLI accept array payloads directly for add and patch flows, without introducing a new DSL.

## TODOS
- [ ] Define accepted JSON array forms for `canvas add`
- [ ] Define accepted JSON array forms for `canvas patch`
- [ ] Keep single-object payloads working as-is
- [ ] Support arrays in inline JSON, file, and stdin paths where applicable
- [ ] Define how patch arrays map to explicit `--id` targets or embedded ids
- [ ] Keep `--json` output explicit about matched, changed, and skipped items
- [ ] Add validation errors for mixed/invalid payload shapes
- [ ] Add help examples for batch add and batch patch
- [ ] Add tests for success, partial failure, and validation failure cases

## NOTES
- User does not want a diagram DSL yet
- JSON is enough; batch JSON is the goal
- `add` should support adding many elements in one payload
- `patch` should support applying many patch entries in one payload
- Prefer one clear contract over magic coercion
- Need to avoid ambiguity between repeated flags and one array payload
- Good to decide whether patch arrays are ordered and whether duplicate ids are allowed
