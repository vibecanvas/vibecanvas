# A30 - theme: remove Tailwind from @vibecanvas/canvas and ship package CSS
[Overview](../BASED.md)

Remove Tailwind coupling from the canvas package and make canvas UI ship its own CSS.

## Context
`@vibecanvas/canvas` is used by frontend, but today frontend CSS scans canvas source with Tailwind `@source`. That couples package styling to the app build. Theme work should move canvas UI to package-owned CSS using shared theme vars.

Relevant files:
- `packages/canvas/package.json`
- `packages/canvas/src/index.ts`
- `packages/canvas/src/components/`
- `packages/canvas/src/new-plugins/`
- `apps/frontend/src/index.css`
- `apps/frontend/vite.config.ts`

## Plan
### 1. Add package-owned stylesheet
Create `packages/canvas/src/styles.css` and import it from the canvas package entry.

### 2. Replace Tailwind utility classes in canvas DOM overlays
Move canvas components to stable semantic class names.

### 3. Stop requiring app-side source scanning
After package CSS is in place, frontend should not need to scan canvas source to style package components.
