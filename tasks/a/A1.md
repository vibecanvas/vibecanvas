# A1 - file: support common CodeMirror languages

## Plan
- Replace the narrow file-extension switch in the file widget's CodeMirror language resolver.
- Prefer CodeMirror's filename-based language metadata so the editor supports common programming languages without a large hand-maintained map.
- Keep plain-text fallback behavior for unknown or unsupported files.

## NOTES
- Category: addition.
- User request: in `packages/canvas/src/components/file/`, make the CodeMirror instance support syntax support for all common programming languages.
- Current implementation uses a small manual switch in `getLanguageExtension.ts` for js/ts/json/css/html/python/sql/rust/markdown/yaml/xml.
- `CodeEditor.tsx` already supports async language loading via `getLanguageExtension(props.path)`, so the main change should stay localized to the resolver and package dependencies.
- Recommended direction: use `LanguageDescription.matchFilename(...)` with `languages` from `@codemirror/language-data`, then `load()` the matched language support and return its extension.
- Repo currently lacks `@codemirror/language-data` and the broader optional language packages that package can dynamically load for many common languages.
