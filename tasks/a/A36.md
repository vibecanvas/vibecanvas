# A36 - theme: centralize canvas style ownership in ThemeService
[Overview](../BASED.md)

ThemeService becomes the single source of truth for canvas style defaults, token lookup, per-tool manual style memory, and theme-driven runtime resolution. Canvas docs keep sparse optional style fields plus base transform scale, and renderers resolve concrete values through ThemeService.

## Context
User requested a style-system refactor centered on `packages/service-theme/src/ThemeService.ts` and `packages/canvas/src/plugins/selection-style-menu/SelectionStyleMenu.plugin.ts`.

Locked requirements from the request:
- ThemeService owns canvas style defaults and theme lookup.
- ThemeService stores last manual selection per tool (`pen`, `text`, `rectangle`, `arrow`, etc.).
- ThemeService exposes lookup tables for style token -> color key / runtime value and notifies subscribers on theme switch.
- Canvas doc style fields stay optional.
- No backward compatibility or migration layer.
- `zElementStyle` must change away from the old mixed numeric shape.
- `headerColor` and `borderColor` are removed.
- `strokeColor` is the canonical stroke/border color field.
- Tokenized style fields:
  - `backgroundColor`
  - `strokeColor`
  - `fontSize`
  - `cornerRadius`
  - `strokeWidth`
- Non-tokenized style fields:
  - `opacity`
  - `strokeStyle`
  - `textAlign`
  - `verticalAlign`
- Text persistence must stop storing runtime `fontSize` / `fontSizePreset` / `lineHeight`; runtime converts tokens to numbers when rendering.
- Base element needs persisted `scaleX` / `scaleY` because some element kinds scale instead of baking geometry.
- Shape2d single-node resize keeps updating geometry (`x/y/w/h`) and resets scale to `1/1`, but grouped or multi-selected shape2d can persist scale.
- Image, pen, and text persist scale.
- Selection style menu must not own style state; ThemeService becomes the style-state source.
- Non-style element data editors are out of scope for this task.

Current drift discovered during planning:
- `ThemeService` currently knows themes + color palette only; it does not own per-tool remembered style state.
- `EditorService` currently owns per-tool remembered selection style values.
- `CanvasRegistryService` definitions currently mix supported controls with hardcoded default values.
- Several serializers infer style back from runtime Konva node values, which fights the new rule that missing style fields should remain missing and be resolved by ThemeService defaults.
- Text still persists `fontSize`, `fontSizePreset`, `textAlign`, `verticalAlign`, and `lineHeight` in `data`.
- Theme tokens only support `gray/red/...` with steps `100/300/500/700/900`, but tests already use tokens like `@red/600`; token support needs to expand to full `100..900` and `gray` must become `base`.

### Locked target model
#### Base element
Persist transform state in base element:
- `x`
- `y`
- `rotation`
- `scaleX`
- `scaleY`

#### Style struct
Target `style` fields, all optional:
- `backgroundColor?: string`
- `strokeColor?: string`
- `strokeWidth?: string`
- `opacity?: number`
- `cornerRadius?: string`
- `strokeStyle?: "solid" | "dashed" | "dotted"`
- `fontSize?: string`
- `textAlign?: "left" | "center" | "right"`
- `verticalAlign?: "top" | "middle" | "bottom"`

#### Text data
Keep text `data` structural, not style-driven:
- keep: `w`, `h`, `text`, `originalText`, `fontFamily`, `link`, `containerId`, `autoResize`
- remove: `fontSize`, `fontSizePreset`, `textAlign`, `verticalAlign`, `lineHeight`
- render/edit code resolves numeric font size and line height at runtime from style + constants

#### Canonical color field usage
- `backgroundColor` is fill.
- `strokeColor` is stroke/border/text color.
- Stroke-only elements (`line`, `arrow`, `pen`, `text`) should stop abusing `backgroundColor` as a fallback stroke field.
- Keep `@transparent` as a special fill token unless implementation proves it is unused.

#### Shape radius ownership
`cornerRadius` belongs in `style`, not `data`.
Current `rect.radius` / `diamond.radius` are unused in runtime code and should be removed unless an audit finds real geometry ownership.
