# A19 - canvas CLI: `add` command

## Plan

None of the existing canvas CLI commands create elements — they all mutate or read what is already there. Add a new `vibecanvas canvas add` subcommand that creates primitive elements (rect, ellipse, diamond, text, line, arrow) inside one selected canvas, mirroring the structure of `cmd.patch.ts` end-to-end.

Shape mirrors `patch`:
- Canvas selector: `--canvas <id>` XOR `--canvas-name <query>`.
- Payload source: `--element <json>` (repeatable) XOR `--elements-file <path>` XOR `--elements-stdin`.
- Common flags: `--db`, `--json`, `--help/-h`.
- Offline path via `createOfflineCanvasState`; online path via `createOnlineCanvasSafeClient` when a local dev server is discovered, same as every other mutation command.

Scope for v1:
- Elements only (no group creation — `group`/`ungroup` already cover that).
- Primitives only: rect, ellipse, diamond, text, line, arrow. Image / pen / widget elements (filetree, terminal, file, iframe-browser) are follow-ups.
- JSON envelope input only — no sugar flags like `--rect x,y,w,h` in v1.

Element factory rules:
- `id`: `crypto.randomUUID()` unless caller passes explicit `id`; reject collisions with existing element/group ids.
- `createdAt = updatedAt = Date.now()`, `bindings = []`, `locked = false` unless overridden.
- `zIndex`: assigned at insert time as `z${padStart(8,'0')}` after max existing — CLI has no RenderOrder plugin to do it later.
- `parentGroupId` must exist in `nextDoc.groups` when non-null.
- Per-type defaults copied from the interactive plugins (Shape2d, Shape1d, Text, pen.constants) so CLI-created elements match drawn ones.
- Do NOT import `packages/functional-core/.../fn.create-element.ts` — it is stale (emits `angle` instead of `rotation`).

Implementation lives in two layers, mirroring patch:
1. `packages/canvas-cmds/src/cmd.add.ts` — `executeCanvasAdd(ctx, input): TCanvasAddSuccess`, exported from `index.ts`. Extract the element-data / style validators currently inlined in `cmd.patch.ts` into a shared file so both cmds reuse them.
2. `packages/api-canvas-cmd/src/contract.ts` + router + `apps/cli/src/plugins/cli/canvas.online-client.ts` — add an `add` procedure so the online safe-client has the new verb.
3. `apps/cli/src/canvas-cli/cmd.add.ts` — thin CLI shim cloned from `cmd.patch.ts`, wired into `apps/cli/src/main.ts`.
4. `renderCanvasAddText` in `packages/canvas-cmds/src/formatters.ts`.

Full design including per-type default tables and file references lives at `~/.claude/plans/foamy-wiggling-lerdorf.md`.
