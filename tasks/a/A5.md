# A5 - canvas CLI: explicit --db path override

## Plan
- Resolve the database path from CLI flags before importing modules that open config, SQLite, or Automerge state at module load.
- Reuse the same explicit path for offline canvas commands and tests so local sandboxes never touch default user data.
- Keep the contract narrow: one explicit db file in, deterministic bootstrap/errors out.

## TODOS

- [x] Introduce an early canvas CLI bootstrap/parser so `--db` is available before importing stateful modules.
- [x] Thread the resolved db path through the offline canvas command open path shared by inspect/query/mutation commands.
- [x] Document precedence and failure behavior for `--db` in help text and JSON errors.
