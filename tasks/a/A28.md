# A28 - theme: add frontend theme picker and quick appearance controls
[Overview](../BASED.md)

Add proper frontend theme controls beyond the basic light/dark toggle.

## Context
Frontend sidebar now has a quick light/dark toggle, but shared theme service already supports more themes such as `sepia` and `graphite`. Users need a way to pick exact themes from the frontend.

Relevant files:
- `apps/frontend/src/feature/sidebar/components/Sidebar.tsx`
- `apps/frontend/src/services/theme.ts`
- `apps/frontend/src/store.ts`
- `packages/service-theme/src/builtins.ts`

## Plan
### 1. Define frontend theme control UX
Keep one fast appearance toggle and add one exact theme picker.

### 2. Wire picker to shared theme service
Read available themes from `ThemeService` instead of hardcoding a frontend-only list.

### 3. Persist selection cleanly
Make sure exact selected theme stays in persisted frontend state.
