# A32 - hosted components: sandboxed Arrow runtime + per-component official packages
[Overview](../BASED.md)

Move hosted canvas widgets to sandboxed Arrow component bundles with builtin defaults and per-slot override support.

## Context
Current hosted UI is hardwired in canvas package:
- `packages/canvas/DESIGN.md`
- `packages/canvas/src/plugins/HostedSolidWidget/HostedSolidWidget.plugin.tsx`
- `packages/canvas/src/plugins/HostedSolidWidget/HostedSolidWidget.mount.tsx`
- `packages/canvas/src/plugins/IframeBrowserWidget/IframeBrowserWidget.plugin.tsx`
- `packages/canvas/src/plugins/ArrowJs/ArrowJs.plugin.ts`
- `packages/canvas/src/services/canvas/interface.ts`
- `packages/canvas/src/runtime.ts`
- `packages/service-automerge/src/types/canvas-doc.types.ts`

Need new architecture:
- trusted canvas host owns Konva node, drag, transform, focus, projection, and CRDT
- Arrow sandbox owns untrusted widget UI
- official widgets are not one shared package; each official widget gets its own package
- later user can load remote replacement without rebuilding app
- builtin official widgets still ship with app
- user can override slots like `filetree` with another implementation

Target package split:
- `packages/hosted-component-sdk`
- `packages/hosted-component-filetree`
- `packages/hosted-component-file`
- `packages/hosted-component-terminal`
- `packages/hosted-component-iframe`

## Plan
### 1. Define hosted component contract
Pin bundle format, manifest, permissions, bridge modules, and slot override rules.

### 2. Build trusted host path in canvas
Add one hosted-component host plugin/service path that mounts Arrow sandbox UI but keeps canvas behavior trusted.

### 3. Split official widgets into per-component packages
Make filetree, file, terminal, and iframe each ship as its own builtin package and register through the same resolver used for remote installs.

### 4. Plan remote install and live swap path
Define fetch, cache, validation, and remount flow so a running app can swap providers without rebuild.
