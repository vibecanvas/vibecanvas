# A25 - canvas: react to live Automerge doc changes without page refresh

## Plan
- Treat live canvas updates as a missing capability, not a sync bug.
- Subscribe canvas runtime to Automerge doc handle change events after initial load.
- Rehydrate or incrementally sync scene state when remote doc changes arrive.
- Add regression coverage for CLI mutation while canvas page is already open.

## NOTES
- Investigation from B19 showed sync already works end to end.
- CLI without `--db` dispatches to the running local server first.
- Server mutation commands call `handle.change(...)` on the server Automerge repo handle.
- Browser Automerge handle receives the change event and exposes patch payloads for the new ellipse add.
- Current failure is render-side only: scene hydrates once on init and does not react to later doc changes.
- `SceneHydratorPlugin` currently loads persisted doc state during init only.
- This work is an addition because realtime canvas scene reaction is missing behavior, not broken sync transport.
