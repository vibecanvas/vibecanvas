# A29 - theme: remember last light and dark theme choices when toggling
[Overview](../BASED.md)

Make the appearance toggle restore the user's last chosen light and dark themes instead of snapping back to plain light/dark.

## Context
Current toggle can switch appearance, but if the user prefers `graphite` for dark or `sepia` for light, a plain toggle should restore those choices instead of losing them.

Relevant files:
- `apps/frontend/src/services/theme.ts`
- `apps/frontend/src/store.ts`
- `apps/frontend/src/feature/sidebar/components/Sidebar.tsx`
- `packages/service-theme/src/ThemeService.ts`
- `packages/service-theme/src/builtins.ts`

## Plan
### 1. Define state model
Track current theme id plus last selected light theme id and last selected dark theme id.

### 2. Update toggle behavior
Appearance toggle should switch to the remembered theme for that appearance bucket.

### 3. Keep direct theme selection in sync
When a user picks a specific theme, update the remembered slot for that appearance.
