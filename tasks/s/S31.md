# S31 - frontend: remove remaining Tailwind-pattern source from app UI
[Overview](../BASED.md)

Frontend still reads like Tailwind even after Tailwind package removal. `apps/frontend/src/index.css` still carries utility-style selectors, and many frontend `.tsx` files still use utility-shaped class strings.

## Context
Canvas package is now moved off Tailwind-shaped source and off the old giant package stylesheet. Frontend still has the same smell.

Quick audit found utility-style classes in files like:
- `apps/frontend/src/App.tsx`
- `apps/frontend/src/index.tsx`
- `apps/frontend/src/pages/welcome.tsx`
- `apps/frontend/src/components/path-picker-dialog.tsx`
- `apps/frontend/src/components/ui/Toast.tsx`
- `apps/frontend/src/components/ui/scroll-area.tsx`
- `apps/frontend/src/feature/sidebar/components/*.tsx`
- `apps/frontend/src/index.css`

## Plan
### 1. Audit real frontend styling boundaries
Decide what global CSS should stay in frontend and what should move next to components.

### 2. Replace utility-like CSS source
Remove Tailwind-pattern selectors from `apps/frontend/src/index.css`.

### 3. Migrate frontend TSX markup
Replace utility-shaped class strings with semantic classes or inline style where simpler.

### 4. Add guard
Add a small regression check so utility-like frontend source does not creep back in.
