# S36 - transform ownership: plugin only renders/dispatches; element plugins own drag + crdt
[Overview](../BASED.md)

Fix transform architecture split: Transform.plugin should only manage transform UI/dispatch, while element plugins like pen own direct drag listeners and CRDT writes.

## Context
Recent migration work moved generic move behavior toward `packages/canvas/src/plugins/transform/Transform.plugin.ts`, but this created an ownership mismatch.

Current problem:
- `Transform.plugin.ts` now owns parts of move lifecycle, proxy drag, and transform callback dispatch.
- element plugins still attach their own runtime listeners in some places, but the intended ownership is unclear.
- pen direct dragging can bypass transform-owned move flow, which causes inconsistent CRDT behavior.
- trying to centralize default move behavior in `CanvasRegistryService` was also wrong: registry should declare hooks/options, not own default runtime behavior.

Desired direction:
- `Transform.plugin.ts`
  - render transformer UI
  - react to anchor pull / drag UI
  - dispatch optional element callbacks (`onMove`, `onResize`, `onRotate`)
  - no CRDT ownership
- element plugins (starting with pen)
  - register tools
  - register element definitions with optional transform callbacks/options
  - attach their own direct drag listeners on their own runtime nodes
  - own CRDT patch policy/history for their own drag behavior
- if an element does not provide move/resize/rotate handlers, transform should not invent default semantic ownership in registry

## Plan
### 1. Clarify transform ownership
- keep transform plugin focused on transformer rendering and callback dispatch only
- remove CRDT ownership from transform plugin move path
- confirm direct-node dragging remains possible for element plugins

### 2. Re-split registry vs runtime behavior
- keep typed transform options and optional callbacks in canvas registry
- ensure registry stays declarative and does not own default transform runtime behavior
- document expected semantics for `onMove`, `onResize`, `onRotate`

### 3. Restore plugin-local drag ownership
- pen plugin should own direct drag listeners and CRDT writes again
- verify shape/image/group plugins follow the same architecture rule or record follow-up tasks where not aligned
- keep transform box independent from plugin-local drag persistence

### 4. Verify behavior and tests
- update affected tests for registry callback API shape
- verify pen direct drag, clone drag, resize, and selection interplay
- verify transform plugin still works for resize/rotate dispatch without owning CRDT
