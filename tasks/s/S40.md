# S40 - canvas: move inline shape text ownership into shape2d
[Overview](../BASED.md)

Inline text for rect/diamond/ellipse should be owned by shape2d data, not by separate text elements.

## Context
Current canvas runtime models shape inline text as a separate `text` element linked by `containerId`. This makes clone, selection, hydration, delete, and ownership boundaries more complex than needed.

Relevant files:
- `packages/service-automerge/src/types/canvas-doc.zod.ts`
- `packages/service-automerge/src/types/canvas-doc.types.ts`
- `packages/canvas/src/plugins/shape2d/Shape2d.plugin.ts`
- `packages/canvas/src/plugins/shape2d/fx.attached-text.ts`
- `packages/canvas/src/plugins/shape2d/fx.to-element.ts`
- `packages/canvas/src/plugins/shape2d/tx.update-node-from-element.ts`
- `packages/canvas/src/plugins/text/Text.plugin.ts`
- `packages/canvas/src/plugins/text/tx.enter-edit-mode.ts`
- `packages/canvas/src/plugins/select/tx.handle-element-pointer-down.ts`
- `packages/canvas/src/services/canvas-registry/CanvasRegistryService.ts`

Target direction:
- free text stays a standalone `text` element owned by the text plugin
- inline text for shape2d becomes part of shape2d persisted data
- inline text runtime rendering may still use `Konva.Text`, but only as a derived child/view detail
- clone and selection should operate on one shape root -> one shape2d element for inline-text shapes

## Plan
### 1. Decide new shape2d data contract
- define inline text payload inside shape2d data
- remove `containerId`-style ownership for shape2d inline text
- list migration expectations for old docs

### 2. Move shape2d runtime ownership
- make shape2d serializers create/read inline text from shape2d data
- keep any runtime text node as internal shape2d implementation detail
- stop treating inline shape text as a standalone canvas element

### 3. Narrow text plugin scope
- keep free-text creation/editing in text plugin
- remove shape2d inline-text ownership from text plugin and attached-text helpers
- update editing entrypoints so shape2d owns inline-text edit flows

### 4. Re-check shared behaviors
- verify clone path is one root shape -> one persisted element for shape2d inline text
- verify selection, delete, hydrate, and history boundaries stay on shape2d
- note any follow-up cleanup tasks if generic clone refactor is still needed after this
