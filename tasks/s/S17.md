# S17 - extract apps/server into apps/cli + shared packages

## Plan
Break the monolithic apps/server into a plugin-based apps/cli that composes shared packages.
apps/server stays as legacy until fully migrated.

Important architectural rule:
- plugins are wiring/lifecycle units
- services are runtime capabilities behind stable interfaces
- plugins must depend on service contracts, not concrete implementations

Example:
- `ApiPlugin` should depend on `IDbService`
- not on `SqliteDbService` directly

This keeps implementations swappable and lets future apps (e.g. `apps/saas`) reuse only the contracts + implementations they need.

Target structure:
```
apps/cli/              -- plugin-based entry point (replaces apps/server)
packages/tapable/      -- extract hook system (SyncHook, AsyncSeriesHook, etc.)
packages/api/          -- business logic / service layer (ORPC contracts + handlers)
packages/db/           -- database (SQLite/Drizzle)
packages/filesystem/   -- filesystem watcher
packages/pty/          -- terminal service
packages/automerge/    -- CRDT repo
```

This enables future apps (e.g. apps/saas) to reuse parts of the stack
without pulling in the full server.

Builds on [E13](../e/E13.md) plugin system design.

## NOTES

### Why
- apps/server is monolithic: DB, PTY, automerge, API, network, CLI all wired in one place
- Can't reuse pieces independently (e.g. saas needs API+DB but not PTY)
- Plugin system (E13) gives the composition model, this task does the extraction

### Next extraction order
1. Define service contracts first (`IDbService`, `IApiService`, etc.)
2. Move concrete implementations behind those contracts into shared packages
3. Add provider plugins for each implementation
4. Add builds for CLI/server/saas-style compositions

### Open decisions / reminders
- `apps/server` remains legacy during migration; avoid big-bang replacement.
- `apps/cli` is the new runtime home for the plugin system.
- Future apps should compose subsets of the stack through service contracts.
- Route/command registries still need to be formalized once service packages land.
