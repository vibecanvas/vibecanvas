# S35 - canvas/core: align fn fx tx file boundaries, injected portals, and callers
[Overview](../BASED.md)

Canvas core drifted into mixed fn/fx/tx file shapes. Several core files read mutable Konva/editor state while still living in `fn.*` files or plain files, `pretext.ts` imports runtime library directly instead of using an injected portal, and callers still assume old render/Konva export patterns.

## Context
User asked to normalize `packages/canvas/src/core` so file names and call signatures match the intended functional-core split:
- `fx.*` files must use `(portal, args)` even when `portal` is empty.
- `fn.*` files must stay logic-only.
- `tx.*` files own writes.
- core must not rely on direct Konva runtime exports from services; use `import type Konva` in core and inject runtime constructors/helpers through `TPortal*`.
- `pretext.ts` must stop importing runtime lib directly from core call sites; caller passes pretext runtime in portal.
- callers across plugins/services must be updated to the new import paths and arg shapes.

Primary audit scope:
- `packages/canvas/src/core/fn.canvas-node-semantics.ts`
- `packages/canvas/src/core/fn.filter-selection.ts`
- `packages/canvas/src/core/fn.get-node-z-index.ts`

## Plan
### 1. Re-classify core files
Decide which core files stay `fn.*`, which move to `fx.*`, which split, and which plain files should disappear.

### 2. Inject dangerous/runtime dependencies
Move Konva runtime checks and pretext runtime usage behind `TPortal*` inputs.

### 3. Update all direct callers in dependency order
Update imports and call signatures from low-level helpers up through plugins and services.

### 4. Validate package state
Run a focused TypeScript check for `packages/canvas` and fix fallout.

## HOW TO CONTINUE
### Main goal of the next pass
Finish `packages/canvas/src/new-plugins` so plugins no longer depend on `SceneService` exposing Konva constructors directly. Use runtime `Konva` where needed or pass constructors through local portals.

### Recommended strategy
1. Fix one plugin area at a time.
2. For each area:
   - make service maps use `scene: SceneService` when the plugin requires `ctx.services.require("scene")`
   - keep local variable names as `scene` too if that reduces confusion
   - replace `render.X` / `portal.render.X` constructor checks with explicit `Konva.X` or injected constructors in local helper portals
   - re-run focused typecheck grep for that area before moving on
3. Avoid bulk regex replacements now. The code is in a half-migrated state and broad replacements already caused some broken call shapes and duplicate inserts.
