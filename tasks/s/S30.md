# S30 - canvas: remove remaining Tailwind-style classes from component TSX
[Overview](../BASED.md)

Canvas package still has Tailwind-shaped class strings in component TSX. Remove them. Some wrappers can die. Some states need semantic CSS or inline style.

## Context
A30 removed Tailwind from `@vibecanvas/canvas` by shipping package CSS, and A31 removed Tailwind from frontend. But canvas still carries a utility-compat layer in `packages/canvas/src/styles.css`, and `packages/canvas/src/components/**/*.tsx` still uses utility-style `class` / `classList` strings.

Search in `packages/canvas/src/components` found 20 `.tsx` files with leftover utility usage. The remaining `SelectionStyleMenu/*.tsx` files are already mostly inline-style driven and are not the main cleanup target.

## Plan
### 1. Kill dead paths and classify leftovers
First find dead component paths, shared wrapper patterns, and true one-off states. Do not restyle dead code.

### 2. Add semantic canvas-owned CSS
Add stable `vc-*` classes in `packages/canvas/src/styles.css` for real component pieces. Use inline style only for true one-off values. Stop adding more utility aliases.

### 3. Migrate overlay UI
Move menu, help, recorder, and toolbar UI off utility strings and onto semantic classes or explicit state attributes.

### 4. Migrate hosted file and filetree UI
Replace widget shells, viewer empty states, list rows, and dialog chrome with package-owned classes.

### 5. Migrate terminal UI
Replace terminal chrome and mount defaults with stable terminal classes and keep token-driven colors.

### 6. Delete dead utility compatibility rules
After TSX migration, remove unused utility selectors from `packages/canvas/src/styles.css` and keep only tokens, resets, and semantic component CSS.

### 7. Add regression check
Keep one smoke/regression check so new utility strings do not creep back in unnoticed.

## NOTES
- Goal is not only “works without app Tailwind”. Goal is “canvas component markup no longer looks like Tailwind and does not depend on utility-compat CSS bloat”.
- Prefer semantic classes like `vc-toolbar`, `vc-terminal-header`, `vc-filetree-row`, not new utility aliases.
- Arbitrary utility values like `w-[min(...)]`, `tracking-[0.2em]`, `z-[9999]`, and `bg-.../..` should move to normal CSS.
- Kobalte state selectors can be styled with normal `[data-*]` selectors on component classes.
- Hosted widget wrappers in file/filetree/terminal look near-identical. Good chance to collapse them into one small shared class or drop extra wrappers.
- `Canvas-old.tsx` has no obvious code references from quick search. Delete if truly dead.
