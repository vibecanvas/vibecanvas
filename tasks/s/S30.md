# S30 - canvas: remove remaining Tailwind-style classes from component TSX
[Overview](../BASED.md)

Canvas package still has Tailwind-shaped class strings in component TSX and Tailwind-pattern selectors in CSS. Remove that whole pattern. Some wrappers can die. Some states need semantic CSS or inline style.

## Context
A30 removed Tailwind from `@vibecanvas/canvas` by shipping package CSS, and A31 removed Tailwind from frontend. But canvas still carries a Tailwind-lookalike utility-compat layer in package CSS, and `packages/canvas/src/components/**/*.tsx` still uses utility-style `class` / `classList` strings.

That means build-time Tailwind is gone, but source still implies Tailwind. Humans and agents can still read the code and think Tailwind is in use.

Search in `packages/canvas/src/components` found 20 `.tsx` files with leftover utility usage. The remaining `SelectionStyleMenu/*.tsx` files are already mostly inline-style driven and are not the main cleanup target.

There is also a follow-up audit question: should canvas keep only a tiny package base stylesheet for tokens/reset while component CSS lives beside each component file, and does `apps/frontend/src/index.css` still need to exist in current shape? If they stay, they should contain owned semantic styling, not pseudo-Tailwind catalogs.

## Plan
### 1. Kill dead paths and classify leftovers
First find dead component paths, shared wrapper patterns, and true one-off states. Do not restyle dead code.

### 2. Decide what CSS files should exist
Audit whether `packages/canvas/src/styles.css` should stay as one file, be split, or be renamed. Same audit for frontend root CSS from A31. Keep files only if they own real package/app styling, not fake utility catalogs.

### 3. Add semantic canvas-owned CSS
Add stable `vc-*` classes for real component pieces. Use inline style only for true one-off values. Stop adding more utility aliases and remove Tailwind-pattern selector names.

### 4. Migrate overlay UI
Move menu, help, recorder, and toolbar UI off utility strings and onto semantic classes or explicit state attributes.

### 5. Migrate hosted file and filetree UI
Replace widget shells, viewer empty states, list rows, and dialog chrome with package-owned classes.

### 6. Migrate terminal UI
Replace terminal chrome and mount defaults with stable terminal classes and keep token-driven colors.

### 7. Delete Tailwind-pattern CSS
After TSX migration, remove utility selectors from `packages/canvas/src/styles.css` and keep only tokens, resets, and semantic component CSS. No `.flex`, `.w-full`, `hover:*`, `dark:*`, `md:*`, bracket-value utility selectors, or other Tailwind-shaped patterns should remain.

### 8. Add regression check
Keep one smoke/regression check so Tailwind-pattern class strings and selectors do not creep back in unnoticed.

## NOTES
- Goal is not only “works without app Tailwind”. Goal is “canvas source no longer looks like Tailwind and does not teach agents the wrong thing”.
- Prefer semantic classes like `vc-toolbar`, `vc-terminal-header`, `vc-filetree-row`, not new utility aliases.
- Remove Tailwind-pattern selector names from CSS too, not just from TSX. Examples to delete: `.flex`, `.w-full`, `.px-2`, `.text-xs`, `.md\:grid-cols-2`, `[class~="bg-card/95"]`, `[class~="hover:bg-accent"]`, `.dark [class~="dark:hover:bg-stone-800"]`.
- Arbitrary utility values like `w-[min(...)]`, `tracking-[0.2em]`, `z-[9999]`, and `bg-.../..` should move to normal semantic CSS.
- Kobalte state selectors can be styled with normal `[data-*]` selectors on component classes.
- Hosted widget wrappers in file/filetree/terminal look near-identical. Good chance to collapse them into one small shared class or drop extra wrappers.
- `Canvas-old.tsx` has no obvious code references from quick search. Delete if truly dead.
- `packages/canvas/src/base.css` is now a tiny package base file for tokens/reset. Component styling should live beside components in local CSS files or inline styles.
- `apps/frontend/src/index.css` may still be valid, but only if it contains owned semantic styling. It should not stay a pseudo-Tailwind dictionary.
